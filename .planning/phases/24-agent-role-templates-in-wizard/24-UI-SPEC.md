## Spacing Scale

All spacing expressed in terminal character cells (cols × rows), not CSS pixels. The ratatui layout engine uses `Constraint::Length(N)` where N is a row count.

| Token | Cell rows | Usage |
|-------|-----------|-------|
| xs | 1 | Hint lines below fields, spacer rows |
| sm | 2 | Footer area height |
| md | 3 | Single-line text input height (field + border) |
| lg | 5 | Three-option radio list (options + 2 border) |
| xl | dynamic | Template list: `templates.len() + 1 (Custom) + 2 (border)` |
| 2xl | 3 | Header area height |
| content | Min(0) | Trailing spacer — absorbs remaining height |

**Horizontal split for template selector:** 45% left (radio list) / 55% right (description preview). Source: CONTEXT.md §Wizard UX flow + RESEARCH.md Pattern 4.

Exceptions: Template section height is computed dynamically (`template_h = templates.len() + 1 + 2`). Worker list: 10 rows (8 templates + Custom + 2 border). Orchestrator list: 6 rows (3 templates + Custom + 2 border).

## Typography

Terminal TUI — no CSS font sizing. Typography expressed as ratatui Style modifiers.

| Role | Modifier | Color | Usage |
|------|----------|-------|-------|
| Body | none (default) | terminal default | Field content, list items, hint lines |
| Label | BOLD | Cyan | Focused header title, section headings in Summary page |
| Muted | none | DarkGray | Unfocused borders, hint text, unselected radio items |
| Accent | none | Cyan | Focused borders, selected radio marker (●), active step title |

**Line height:** 1 row per line of text. No configurable line spacing in terminal TUI.

**Source:** Codebase scan — `wizard.rs` style constants confirmed as `Color::Cyan` (focus), `Color::DarkGray` (unfocused), `Color::Red` (error), `Color::Green` (new agent tag), `Color::Yellow` (existing agent tag).

## Color

| Role | ANSI Color | Usage |
|------|------------|-------|
| Dominant (60%) | terminal default background | Page background, paragraph body, all unfocused surfaces |
| Secondary (30%) | DarkGray | Unfocused field borders, hint text, unselected radio markers (○) |

4. Template preview panel border when template field is focused
5. Agent fleet diagram box (Orchestrator box lines)
6. Summary section headers (Orchestrator / Workers labels)

## Interaction Contract

### Template selector navigation

| Key | Action |
|-----|--------|
| ↑ / ↓ | Move highlight up/down through template list (preview updates live) |
| Enter / Tab | Confirm selection — apply auto-fill, advance focus to Provider field |
| Esc | Back to Name field (no auto-fill applied) |

**Preview updates on each ↑/↓ keystroke.** Auto-fill fires only on Enter/Tab confirmation.

### Auto-fill behavior on template confirmation

When a template is confirmed (Enter/Tab on Template field):
1. Name field: Always overwrite with template slug (e.g. `coder`, `qa-engineer`). **Exception:** if user typed a custom name before reaching Template field, it is overwritten. Document this as intentional — user can edit Name afterward.
2. Provider field: Set to `template.default_provider`
3. Model field: Set index to first match of `template.claude_model` or `template.gemini_model` in `ModelSelector::options_for(provider)`; fallback to index 0
4. Description field: Set to `template.description` text (cursor positioned at end)
5. `draft.routing_hints`: Set to `Some(template.routing_hints.to_vec())`

**Custom selection behavior:** When `template_index == templates.len()` (the Custom sentinel):
- Name, Provider, Model, Description fields all cleared to empty/defaults
- `draft.routing_hints` set to `None`

All auto-filled fields remain editable after template application. Source: CONTEXT.md §Wizard UX flow.

### Footer hint update for Template field

Add new branch to `agent_footer` closure for `AgentField::Template`:

```
↑↓: select template   Enter/Tab: apply   Esc: back   Ctrl+C: cancel
```
