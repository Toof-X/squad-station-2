## New Surfaces in Phase 17

Phase 17 introduces two interaction surfaces not present in Phase 16:

1. **Re-init prompt** (INIT-05) — A TUI screen shown when `squad.yml` already exists. User chooses one of three options: overwrite, add agents, or abort.
2. **Squad.yml write confirmation** (INIT-04) — Stdout feedback after the wizard completes and squad.yml is written to disk.

All other visual elements (wizard pages, field rendering, error messages, key bindings) are inherited from Phase 16 unchanged.

## Copywriting Contract

### Phase 17 — New Copy Elements Only

#### Re-Init Prompt (INIT-05)

Shown when `squad.yml` exists at the target path before the wizard runs.

| Element | Copy |
|---------|------|
| Block title | ` Existing Configuration Found ` |
| Body line 1 | `  squad.yml already exists for this project.` |
| Body line 2 | `  What would you like to do?` |
| Option 1 (overwrite) | `  [ ] Overwrite — replace existing configuration with a new wizard run` |
| Option 2 (add agents) | `  [ ] Add agents — keep existing config and append new agents via wizard` |
| Option 3 (abort) | `  [ ] Abort — leave squad.yml unchanged and exit` |
| Selected option marker | `  [x]` replaces `  [ ]` on the focused row |
| Footer hint | `↑↓: select   Enter: confirm   Ctrl+C: cancel` |

#### Squad.yml Write Confirmation (INIT-04)

Printed to stdout after terminal is restored, before agent registration begins. These are plain `println!()` calls — not ratatui widgets.

| Element | Copy |
|---------|------|
| Success line | `squad.yml written to {path}` |
| Proceeding line | `Starting agent registration...` |

Note: The path in the success line is the absolute or relative path where the file was written (e.g., `./squad.yml`). Use `display()` on the `PathBuf` to format it.

#### Cancellation / Exit — Phase 17 Additions

| Trigger | Output | Copy |
|---------|--------|------|
| Abort selected on re-init prompt | Stdout after terminal restore | `Existing squad.yml unchanged. Exiting.` |
| Ctrl+C on re-init prompt | Stdout after terminal restore | `Init cancelled.` (matches Phase 16 wizard cancel copy) |

#### Error States — Phase 17 Additions

These are stdout messages printed after terminal restore when squad.yml write fails.

| Condition | Copy |
|-----------|------|
| squad.yml write fails (I/O error) | `Error: failed to write squad.yml — {error message}` |
| squad.yml write fails, file exists and write denied | `Error: cannot write squad.yml — {error message}. Check file permissions.` |

Note: Use `anyhow::bail!` or `return Err(...)` to propagate the error to the caller. The calling code in `init.rs` prints the error via the standard error path — do not add custom println! for errors already surfaced by anyhow.

#### Empty State

Not applicable in Phase 17. The re-init prompt always has exactly 3 options — no empty list state is possible.

#### Destructive Actions

| Action | Trigger | Confirmation Approach |
|--------|---------|----------------------|
| Overwrite squad.yml | User selects "Overwrite" and presses Enter on re-init prompt | The re-init prompt IS the confirmation. No second confirmation dialog. The option text makes the consequence explicit: "replace existing configuration with a new wizard run". |

The overwrite action is confirmed by the explicit selection + Enter combination on the re-init prompt. No additional "Are you sure?" dialog is required because the menu option copy already states the consequence clearly.

## Interaction Contract — Phase 17 Additions

### Re-Init Prompt Key Bindings

| Key | Action |
|-----|--------|
| `↑` or `k` | Move selection to previous option (wraps: abort → add agents → overwrite) |
| `↓` or `j` | Move selection to next option (wraps: overwrite → add agents → abort) |
| `Enter` | Confirm selected option and proceed |
| `Esc` | Treat as abort — same outcome as selecting Abort and pressing Enter |
| `Ctrl+C` | Cancel entirely — same outcome as Ctrl+C in the wizard; stdout: "Init cancelled." |

Default selection on prompt open: `Abort` (index 2, safest default — no data loss without deliberate choice).

### Re-Init Flow Outcomes

| Selection | Outcome |
|-----------|---------|
| Overwrite | Terminal restored → wizard runs from scratch → squad.yml overwritten → agent registration proceeds |
| Add agents | Terminal restored → wizard runs with agent-only pages (skip project name / agent count intro if possible, else run full wizard and merge) → squad.yml updated with appended agents → agent registration proceeds |
| Abort | Terminal restored → stdout: "Existing squad.yml unchanged. Exiting." → `return Ok(())` |

Note on "Add agents" implementation: If merging wizard output with existing config is complex, a full wizard run followed by appending the new agents to the existing `agents:` list in squad.yml is acceptable. The key constraint is that existing agent entries are not removed.

### Squad.yml Write Sequence (INIT-04)

After the wizard returns `Ok(Some(WizardResult))`:

1. Serialize `WizardResult` to squad.yml YAML format.
2. Write to `config_path` (the same PathBuf passed to `init::run()`).
3. Print stdout confirmation: `squad.yml written to {path}`.
4. Fall through to `config::load_config()` to continue with agent registration.

The write must complete before `load_config` is called. If write fails, bail with error — do not proceed to agent registration.

### Page Structure for Re-Init Prompt

The re-init prompt uses the same 3-zone layout as the wizard pages from Phase 16:

```
+-------------------------------------------------+
| Header — 3 rows: "Existing Configuration Found" |  Constraint::Length(3)
+-------------------------------------------------+
| Content — expands to fill                       |  Constraint::Min(10)
|   [re-init menu rendered here]                  |
+-------------------------------------------------+
| Footer — 2 rows: key hints                      |  Constraint::Length(2)
+-------------------------------------------------+
```

The header uses `Color::Cyan` + `Modifier::BOLD` (matching Phase 16 heading style).
The content area renders the 3-option menu using a `List` widget with `ListItem` rows.
The footer renders: `↑↓: select   Enter: confirm   Ctrl+C: cancel`.

## Layout Contract

### Re-Init Prompt Menu Layout

```
Content area:
  +----------------------------------------------+
  |  Existing Configuration Found                |   Block with Borders::ALL
  |                                              |
  |    squad.yml already exists.                |
  |    What would you like to do?               |
  |                                              |
```
