1. **tmux.rs** — Add a new public function `list_live_session_names()`:
   ```rust
   pub fn list_live_session_names() -> Vec<String> {
       let output = Command::new("tmux")
           .args(["list-sessions", "-F", "#{session_name}"])
           .output()
           .unwrap_or_default();
       String::from_utf8_lossy(&output.stdout)
           .lines()
           .map(str::trim)
           .filter(|s| !s.is_empty())
           .map(String::from)
           .collect()
   }
   ```
   Also add a private arg builder `list_sessions_args()` for testability (follows existing pattern from send_keys_args, enter_args, launch_args):
   ```rust
   fn list_sessions_args() -> Vec<String> {
       vec!["list-sessions".into(), "-F".into(), "#{session_name}".into()]
   }
   ```
   Add arg builders for `kill_window_args`, `new_window_args`, `split_window_args`, `select_layout_args` used by the view command, all as private fns with unit tests.

2. **tmux.rs** — Add public functions for view command:
   - `kill_window(window_name: &str) -> Result<()>` — runs `tmux kill-window -t {window_name}`, ignores errors (idempotent teardown)
   - `create_view_window(window_name: &str, sessions: &[String]) -> Result<()>` — creates new window with first session showing `tmux attach-session -t {first_session}`, splits panes for remaining sessions each running `tmux attach-session -t {session}`, applies `select-layout tiled`

3. **commands/view.rs** — Full implementation of VIEW-04:
   - Load config, resolve DB path, connect
   - Fetch agents via `db::agents::list_agents()`
   - Get live tmux sessions via `tmux::list_live_session_names()`
   - Filter: keep only agents whose name appears in live sessions (skip dead agents per user decision)
   - Include orchestrator in the list (all sessions including orchestrator per user decision)
   - If no live sessions: print "No live agent sessions to display." and return Ok(())
   - Call `tmux::kill_window("squad-view")` for idempotency
   - Call `tmux::create_view_window("squad-view", &live_agent_names)`
   - Print confirmation: "Created squad-view with {n} panes" (or JSON equivalent if --json)
   - `view` command accepts `json: bool` parameter for consistency with other commands

   The view command uses the CONTEXT.md approach: each pane runs `tmux attach-session -t {agent_name}` to show live terminal content. This is the established pattern for multi-pane monitoring per RESEARCH.md (Option B).

   When passing the command to `split-window`, use the format:
   `tmux split-window -t squad-view "tmux attach-session -t {session_name}"`

   `tmux new-window -n squad-view "tmux attach-session -t {first_session}"`
