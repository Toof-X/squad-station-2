Create `src/commands/diagram.rs` with:

1. **Imports:**
   ```rust
   use crate::commands::helpers::{colorize_agent_status, pad_colored};
   use crate::db::agents::Agent;
   use owo_colors::OwoColorize;
   use owo_colors::Stream;
   ```

2. **Public function signature:** `pub fn print_diagram(agents: &[Agent])`

3. **Logic flow:**
   - Early return if `agents` is empty
   - Partition agents: `orchestrators` = agents where `role == "orchestrator"`, `workers` = rest
   - Print `"\nAgent Fleet:"`
   - For each orchestrator, call `render_agent_box(agent, true)` to get a `Vec<String>` of lines
   - Center the orchestrator box output (left-pad each line so the box is roughly centered relative to the worker row, or just print flush-left -- flush-left is acceptable)
   - If workers are non-empty, render an arrow row: for each worker box column midpoint, place `│` on one line and `▼` on the next line
   - Render workers in a horizontal row with 2-space gaps. When cumulative width + next box width + gap > 80, wrap to a new row (with a new arrow row above each wrapped row)

4. **Private function `render_agent_box(agent: &Agent, is_orchestrator: bool) -> Vec<String>`:**
   - Build content lines:
     - If `is_orchestrator`: first line = `"ORCHESTRATOR"` rendered with `.if_supports_color(Stream::Stdout, |s| s.bold())`
     - Agent name line: `agent.name`
     - Tool+model line: if `agent.model` is `Some(m)` and not empty, format as `"tool: {tool}  model: {m}"`; if `None` or empty, format as `"tool: {tool}"`
     - Status line: `[idle]`, `[busy]`, `[dead]` -- use `colorize_agent_status(&agent.status)` for the colored version
   - Compute raw content lines (without ANSI codes) for width measurement
   - Box width = max raw content line length + 4 (for `"│ "` left border + `" │"` right border)
   - Build lines:
     - Top: `format!("┌{}┐", "─".repeat(box_width - 2))`
     - For each content line: `format!("│ {} │", pad_colored(raw_line, colored_line, box_width - 4))`
     - Bottom: `format!("└{}┘", "─".repeat(box_width - 2))`
   - Return the `Vec<String>`

5. **Private function `render_arrow_row(worker_boxes: &[Vec<String>], gap: usize) -> Vec<String>`:**
   - Compute midpoint X position for each worker box within the row
   - Build two lines: one with `│` at each midpoint, one with `▼` at each midpoint, rest spaces
   - Return the two lines as `Vec<String>`

6. **Register module:** Add `pub mod diagram;` to `src/commands/mod.rs` in alphabetical order (between `close` and `context` -- wait, between `close` and `context` is wrong. Alphabetically: `clean`, `close`, `context`, `diagram`, `freeze`... so `diagram` goes between `context` and `freeze`).
