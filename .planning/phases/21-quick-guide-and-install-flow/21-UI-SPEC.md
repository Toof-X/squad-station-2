# Phase 21 — UI Design Contract

> Visual and interaction contract for the quick guide TUI page and install auto-launch.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> NOTE: This is a terminal UI (ratatui 0.30 + crossterm 0.29), not a web frontend.
> Web-centric design system concepts (shadcn, Tailwind, CSS tokens, hex colors) do not apply.
> All rendering is ANSI terminal color + ratatui widgets on a monospace character grid.

## Spacing Scale

Terminal spacing is measured in character rows and columns, not pixels. The following table maps logical spacing roles to ratatui `Constraint::Length` values (in terminal rows).

| Token | Rows | Usage |
|-------|------|-------|
| blank | 1 | Empty row separating content regions (blank line between header and steps) |
| section | 1 | Single-row content zones (header line, hint bar, version line) |
| content | Min(0) | Expands to fill remaining rows (steps body, commands table) |

Exceptions: none. All layout uses `Constraint::Length(1)` for fixed rows and `Constraint::Min(0)` for expandable content — matching the existing `draw_welcome()` pattern.

Source: existing `draw_welcome()` layout in `src/commands/welcome.rs`.

## Typography

Terminal typography is character-based. "Size" is a ratatui `PixelSize` or plain text. "Weight" is a ratatui `Modifier`.

| Role | Representation | Modifier | Alignment | Source |
|------|---------------|----------|-----------|--------|
| Display title (title page only) | `BigText` with `PixelSize::HalfHeight` | `Style::fg(Color::Red)` | Centered | Existing `draw_welcome()` |
| Section header (guide page) | Plain `Paragraph` text | none | Centered | CONTEXT.md: "Centered header line" |
| Body / steps | Plain `Paragraph` text | none | Left (default) | CONTEXT.md: "plain text with breathing room" |
| Commands list | Plain `Paragraph` text | `Modifier::BOLD` | Left | Existing `draw_welcome()` |
| Hint bar | Plain `Paragraph` text | `Modifier::DIM` | Left | Existing `draw_welcome()` |
| Footer line | Plain `Paragraph` text | `Modifier::DIM` | Left | CONTEXT.md: footer consistent with existing screen |

## Color

Terminal colors, not hex values. Uses ratatui `Color` enum and `Modifier` enum.

| Role | Value | Usage | Source |
|------|-------|-------|--------|
| Dominant (60%) | `Color::Reset` (terminal default background) | All surfaces — title page, guide page | Existing `draw_welcome()` |
| Secondary (30%) | `Color::Reset` (terminal default foreground) | Body text, commands, steps | Existing `draw_welcome()` |
| Accent (10%) | `Color::Red` | BigText title on title page only | Existing `draw_welcome()`: `Style::default().fg(Color::Red)` |
| Dim (hint bar, footer) | `Modifier::DIM` | Hint bar text on both pages; footer line on guide page | Existing `draw_welcome()` |

Accent reserved for: BigText pixel-font title widget on the title page only. Not used on the guide page. Not used for body text, steps, or the hint bar.

## Install Auto-Launch (Non-Visual Contract)

The install scripts have no TUI or visual output beyond existing success messages. The auto-launch is a behavioral contract, not a visual one.

| Script | Guard | Exec method | Placement | Source |
|--------|-------|-------------|-----------|--------|
| `npm-package/bin/run.js` | `if (process.stdout.isTTY)` | `spawnSync(destPath, [], { stdio: 'inherit' })` | Bottom of `install()` after scaffoldProject() and "Next steps" output | CONTEXT.md locked |
| `install.sh` | `if [ -t 1 ]; then` | `exec "${INSTALL_DIR}/squad-station"` | After final echo statements, before script exit | CONTEXT.md locked |

Silent degradation: Both guards simply do not fire in non-interactive environments (CI, pipes). No additional output, no error messages.

Source: CONTEXT.md locked decisions (INSTALL-01, INSTALL-02, INSTALL-03).
