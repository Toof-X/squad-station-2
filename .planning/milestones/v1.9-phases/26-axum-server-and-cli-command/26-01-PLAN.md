7. **StatusResponse struct + api_status handler:**
```rust
#[derive(Serialize)]
struct StatusResponse {
    project: String,
    agents: usize,
    uptime_secs: u64,
    version: String,
}

async fn api_status(State(state): State<AppState>) -> Json<StatusResponse> {
    let agent_count = match &state.db {
        Some(pool) => crate::db::agents::list_agents(pool).await.map(|a| a.len()).unwrap_or(0),
        None => 0,
    };
    Json(StatusResponse {
        project: state.project_name.clone(),
        agents: agent_count,
        uptime_secs: state.started_at.elapsed().as_secs(),
        version: env!("CARGO_PKG_VERSION").to_string(),
    })
}
```

8. **pub async fn run(port: Option<u16>, no_open: bool) -> anyhow::Result<()>:**
   - Load config via `crate::config::load_config(Path::new(crate::config::DEFAULT_CONFIG_FILE))?`
   - Resolve DB path via `crate::config::resolve_db_path(&config)?`
   - Connect read-only pool: `crate::db::connect_readonly(&db_path).await` — if error, print warning, continue with `None` (same as spike pattern)
   - Extract project name from config (use `config.project` or fallback to "unknown")
   - Call `bind_listener(port)` to get TcpListener
   - Get actual port from `listener.local_addr()?.port()`
   - Print: `"Squad Station browser at http://127.0.0.1:{actual_port}"`
   - If `!no_open`: call `open::that(&url)`, print warning on error (not fatal)
   - Build router: `.route("/api/status", get(api_status)).route("/ws", get(ws_handler)).nest_service("/", ServeEmbed::<FrontendAssets>::new())` — explicit routes FIRST, then SPA fallback LAST
   - Create AppState with db, project_name, started_at
   - Router `.with_state(state)`
   - `axum::serve(listener, app).with_graceful_shutdown(shutdown_signal()).await?`
   - Print "Server stopped." and return Ok

**IMPORTANT anti-patterns to avoid:**
- Do NOT share the single-writer pool — use connect_readonly
- Do NOT hardcode port in URL — always use listener.local_addr()
- Route ordering: /api/status and /ws BEFORE nest_service("/")
- bind_listener must handle the "explicit port errors, default port falls back" distinction per CONTEXT.md
