**Step 1: Install Tailwind CSS v4 in web/:**
```bash
cd web && npm install tailwindcss @tailwindcss/vite
```

**Step 2: Update web/vite.config.ts — add Tailwind plugin:**
```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [
    react(),
    tailwindcss(),
  ],
})
```

**Step 3: Update web/src/index.css — Tailwind v4 import:**
```css
@import "tailwindcss";
```
No `@tailwind base/components/utilities` directives (that's v3). No postcss.config.js or tailwind.config.js (v4 uses Vite plugin).

**Step 4: Create web/src/components/ directory and ConnectionStatus.tsx:**

A component that:
- Creates a WebSocket connection to `ws://${window.location.host}/ws` on mount
- Tracks connection state: "connecting" | "connected" | "disconnected"
- On open: set "connected"
- On close/error: set "disconnected", attempt reconnect after 3 seconds
- Renders a small status indicator:
  - Connected: green dot + "Connected" text
  - Disconnected: red dot + "Disconnected" text
  - Connecting: yellow dot + "Connecting..." text
- Use Tailwind utility classes for styling (flex, items-center, gap, rounded-full, bg-green-500, etc.)
- Clean up WebSocket on unmount (useEffect cleanup)

**Step 5: Create web/src/components/StatusBar.tsx:**

A component that:
- Fetches `/api/status` on mount using `fetch('/api/status')`
- Stores response in state: `{ project: string, agents: number, uptime_secs: number, version: string } | null`
- Re-fetches every 10 seconds (setInterval)
- Renders a horizontal bar showing: project name, agent count, uptime (formatted as "Xm Ys"), version
- Use Tailwind classes for styling (flex, justify-between, px-4, py-2, bg-gray-800, text-white, text-sm, etc.)
- Show "Loading..." while fetching
- Clean up interval on unmount

**Step 6: Update web/src/App.tsx:**

Restructure the app layout:
1. Import ConnectionStatus and StatusBar components
2. Wrap everything in a flex column container (h-screen, flex, flex-col)
3. Top bar: StatusBar component + ConnectionStatus indicator (flex row, justify-between, items-center)
4. Main area: React Flow component (flex-1, takes remaining height)
5. Keep the existing React Flow nodes/edges (Orchestrator + Worker 1 + Worker 2)
6. Apply dark background to the overall container (bg-gray-900, text-white) — gives a professional dashboard feel
7. React Flow should have its own background (the React Flow default is fine)

**Styling notes:**
- Use a dark theme base (bg-gray-900, text-gray-100) — this is the design system foundation for Phase 28
- The status bar should feel like a header/toolbar
- The connection status should be compact (small dot + text) in the top-right area
- Keep it clean and minimal — Phase 28 will build the full polished UI
