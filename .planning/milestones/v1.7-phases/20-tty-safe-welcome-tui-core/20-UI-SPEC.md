# Phase 20 тАФ UI Design Contract

> Visual and interaction contract for the TTY-Safe Welcome TUI Core.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

**Note on design system context:** This phase renders inside a terminal emulator using ratatui,
not a web browser. There is no CSS, no shadcn, no HTML. All visual specifications are expressed
in terminal rendering terms: ratatui `Constraint` values, `Style` attributes, and character-cell
layout. Where the template asks for px values, character-cell equivalents are given instead.

## Spacing Scale

Terminal spacing is expressed in character rows/columns, not pixels.
The 8-point scale maps as follows (1 terminal row тЙИ 16-20px at typical font sizes):

| Token | Terminal Units | Ratatui Constraint | Usage |
|-------|---------------|-------------------|-------|
| xs | 1 col padding | `Constraint::Length(1)` | Inline left/right padding inside widgets |
| sm | 1 row | `Constraint::Length(1)` | Single-row elements (version, tagline, hint bar) |
| md | 2 rows | `Constraint::Length(2)` | Spacer between content sections |
| lg | 4 rows | `Constraint::Length(4)` | BigText title (HalfHeight pixel rows per glyph) |
| xl | 4тАУ8 rows | `Constraint::Length(8)` | Full BigText title block (HalfHeight ├Ч 2 glyph rows) |
| min | fills remaining | `Constraint::Min(0)` | Commands table тАФ expands to fill available space |
| hint | 1 row fixed | `Constraint::Length(1)` | Hint bar тАФ always pinned to bottom |

Exceptions:
- BigText title height is not fixed тАФ it depends on `PixelSize::HalfHeight` rendering which
  consumes 4 terminal rows per glyph row. "SQUAD-STATION" occupies 1 glyph row ├Ч 4 terminal
  rows = **4 rows minimum**. Allocate `Constraint::Length(4)` for the title area.
- No horizontal padding tokens тАФ all horizontal centering is delegated to tui-big-text's
  `.centered()` builder method and ratatui `Alignment::Center`.

Source: RESEARCH.md Layout Structure, Architecture Patterns.

## Typography

Terminal rendering does not allow font size or weight selection тАФ the terminal emulator controls
the physical font. ratatui controls only `Style` attributes: foreground color, `Modifier::BOLD`,
`Modifier::DIM`, and `Modifier::REVERSED`.

| Role | Ratatui Modifier | Color | Usage |
|------|-----------------|-------|-------|
| Display (title) | `Modifier::BOLD`, tui-big-text pixel font | `Color::Red` | BigText SQUAD-STATION title only |
| Heading | `Modifier::BOLD` | `Color::White` (terminal default) | Section labels (e.g., "Commands:") |
| Body | none (regular) | terminal default | Version string, tagline, command descriptions |
| Hint / dim | `Modifier::DIM` | terminal default | Hint bar text |

Line height: 1 terminal row per text row тАФ not configurable; ratatui renders one line per `Line`.

Source: `src/commands/ui.rs` existing `Style` patterns, CONTEXT.md color discretion note.

## Color

| Role | ratatui Value | Usage |
|------|--------------|-------|
| Dominant (60%) | `Color::Reset` (terminal background) | Full AlternateScreen background тАФ inherits user's terminal theme |
| Secondary (30%) | `Color::Reset` (no border fill) | No secondary panel color тАФ welcome screen is single-pane, no borders |
| Accent (10%) | `Color::Red` | BigText SQUAD-STATION title only |
| Status indicators | `Color::Green` / `Color::Yellow` / `Color::Red` | Not used in welcome TUI (carried forward from ui.rs for reference) |

Accent (`Color::Red`) reserved for: **BigText title widget only**. No other element uses red.

Color rationale:
- Red accent for SQUAD-STATION title matches the existing `print_welcome()` pattern
  (`ASCII_ART.if_supports_color(Stream::Stdout, |s| s.red())`) and `ui.rs` title bar
  (`Style::default().fg(Color::Red).add_modifier(Modifier::BOLD)`).
- No border colors are used on the welcome screen тАФ it is a borderless, full-screen layout.
  The existing TUI dashboard (`ui.rs`) uses `Color::Cyan` for focused panel borders, but the
  welcome TUI has no panels.
- Background uses `Color::Reset` so the welcome TUI respects the user's terminal color scheme
  (dark or light themes work without forcing a black background).

Source: CONTEXT.md Claude's Discretion (color scheme), `src/commands/ui.rs` line 184,
`src/commands/welcome.rs` line 47.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (no squad.yml) | `Enter: Set up` |
| Primary CTA (squad.yml exists) | `Enter: Open dashboard` |
| Quit key label | `Q: Quit` |
| Countdown display | `auto-exit {N}s` (live, ticks each second from 5 to 1, then exits) |
| Full hint bar (no squad.yml) | `Enter: Set up  Q: Quit  auto-exit {N}s` |
| Full hint bar (squad.yml) | `Enter: Open dashboard  Q: Quit  auto-exit {N}s` |
| Tagline | `Multi-agent orchestration for AI coding` |
| Version display | `v{semver}` (e.g., `v0.5.3`) тАФ sourced from `env!("CARGO_PKG_VERSION")` |
| Non-TTY static fallback line | `Get started: squad-station init` (preserved from current `print_welcome()`) |

Empty state: Not applicable тАФ the welcome screen has no empty state. The commands table
always shows all 11 subcommands.

Error state: The welcome TUI does not display errors. On any setup/teardown failure,
propagate `anyhow::Result` to the caller (main.rs). The terminal will print the error
via the standard `?` operator before returning to the shell prompt.

Destructive actions: None in this phase. Q/Esc closes silently with no confirmation.
Timeout closes silently with no confirmation.

## Interaction Contract

| Trigger | Condition | Action |
|---------|-----------|--------|
| `Enter` key | no `squad.yml` found | Close TUI, launch `commands::init::run()` |
| `Enter` key | `squad.yml` exists | Close TUI, launch `commands::ui::run()` |
| `Q` key | any | Close TUI, return to shell (no routing) |
| `Esc` key | any | Close TUI, return to shell (no routing) |
| 5-second timeout | no keypress received | Close TUI, return to shell (no routing, same as Q) |
| Non-TTY stdout | at entry point | Skip TUI, call `print_welcome()` static fallback |
| Any other key | countdown active | Ignore key, continue countdown |

Countdown tick behavior:
- Poll with `remaining.min(Duration::from_secs(1))` тАФ updates display at least once per second
- Hint bar shows `auto-exit 5s`, `auto-exit 4s`, `auto-exit 3s`, `auto-exit 2s`, `auto-exit 1s`
- At zero, exit silently without any routing action

State machine (single-page, no navigation in Phase 20):

```
[Entry] тЖТ TTY check
    тФЬтФА not TTY тЖТ print_welcome() тЖТ exit
    тФФтФА is TTY  тЖТ open AlternateScreen
                    тЖТ draw welcome TUI
                    тЖТ event loop (countdown + keypress)
                        тФЬтФА Enter (no config) тЖТ restore тЖТ launch init wizard
                        тФЬтФА Enter (config)    тЖТ restore тЖТ launch ui dashboard
                        тФЬтФА Q / Esc          тЖТ restore тЖТ return to shell
                        тФФтФА timeout           тЖТ restore тЖТ return to shell
```

## tui-big-text Widget Specification

| Property | Value | Rationale |
|----------|-------|-----------|
| `PixelSize` | `HalfHeight` | 4 rows per glyph row vs 8 for `Full`; "SQUAD-STATION" fits in 4 rows leaving room for version, tagline, commands, hint bar on a 24-row terminal |
| Lines | `["SQUAD-STATION"]` | Single line string (13 chars ├Ч 8 cols = 104 cols wide; at `HalfHeight` horizontal size is unchanged тАФ centered by `.centered()`) |
| Style | `Style::default().fg(Color::Red)` | Red accent consistent with existing `print_welcome()` and `ui.rs` title bar |
| Alignment | `.centered()` builder method | Horizontal center in allocated area |
| Allocated area | `Constraint::Length(4)` | Minimum 4 rows for one glyph row at `HalfHeight` |

Narrow terminal fallback: If the terminal is fewer than 80 columns wide, ratatui clips the
BigText widget to the available area. The text will be partially visible but will not panic.
No responsive fallback is implemented in Phase 20 тАФ this is acceptable for the CLI audience.

Source: RESEARCH.md Pattern 4, Open Questions item 1, Pitfall 5.

## Terminal Setup / Teardown Contract

| Step | Action | ratatui/crossterm Call |
|------|--------|----------------------|
| Entry | Install panic hook | `std::panic::set_hook` with `disable_raw_mode` + `LeaveAlternateScreen` |
| Entry | Enable raw mode | `crossterm::terminal::enable_raw_mode()` |
| Entry | Open AlternateScreen | `execute!(stdout, EnterAlternateScreen)` |
| Exit (any path) | Disable raw mode | `crossterm::terminal::disable_raw_mode()` |
| Exit (any path) | Leave AlternateScreen | `execute!(terminal.backend_mut(), LeaveAlternateScreen)` |
| Exit (any path) | Restore cursor | `terminal.show_cursor()` |
| Post-exit | Restore default panic hook | `std::panic::take_hook()` |

After `LeaveAlternateScreen` returns, the terminal is in normal mode. The routing action
(init wizard or dashboard) is invoked after restore_terminal() completes. No output is
printed between restore and routing тАФ scrollback contains no artifact from the welcome TUI.

Source: `src/commands/ui.rs` lines 145тАУ158, 283тАУ289, CONTEXT.md Screen mode decision.
