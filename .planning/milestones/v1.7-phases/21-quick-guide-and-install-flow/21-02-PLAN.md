**Part A — npm-package/bin/run.js changes:**

**A1. Make installBinary() return destPath:**

The `destPath` variable is declared at line 74: `var destPath = path.join(installDir, 'squad-station');`

There are two exit paths from `installBinary()`:
1. Early return on line 82 (binary already installed, correct version): Change `return;` to `return destPath;`
2. End of function after chmod (line 110-111 area): Add `return destPath;` as the last line of the function

**A2. Capture return value in install():**

Change line 31 from:
```javascript

```
To:
```javascript

```

**A3. Add TTY-guarded auto-launch at the bottom of install():**

After the final `console.log` line in install() (the one with "3. Run  ..." at approximately line 41), before the closing `}` of install(), add:

```javascript
  // Auto-launch welcome TUI in interactive terminals
  if (process.stdout.isTTY) {
    spawnSync(destPath, [], { stdio: 'inherit' });
  }
```

`spawnSync` is already imported at line 6: `const { spawnSync } = require('child_process');`

No new imports needed. The `destPath` is the full absolute path to the installed binary, resolving any PATH uncertainty.

**Part B — install.sh changes:**

**B1. Add TTY-guarded exec at the end of install.sh:**

After the final `fi` block (the FALLBACK PATH warning around line 67), at the very end of the file, add:

```sh

# Auto-launch welcome TUI in interactive terminals
if [ -t 1 ]; then
  exec "${INSTALL_DIR}/squad-station"
fi
```

Key details:
- `[ -t 1 ]` tests if file descriptor 1 (stdout) is a terminal
- `exec` replaces the shell process with the binary (clean handoff, no extra process)
- The `trap 'rm -f "$TMPFILE"' EXIT` fires when `exec` is called, but by this point `TMPFILE` has already been `mv`'d to `${INSTALL_DIR}/squad-station` (line 53), so `rm -f` on the no-longer-existing temp file is a harmless no-op
- No additional CI env var guards (`$CI`, `$GITHUB_ACTIONS`, etc.) — TTY check alone is sufficient per locked decision
- The `exec` line is only reached in interactive terminals; in pipes/CI, the `if` does not fire and the script exits normally
