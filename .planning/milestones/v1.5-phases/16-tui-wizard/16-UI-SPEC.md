## Spacing Scale

In ratatui, spacing is expressed in terminal cell rows/columns via `Constraint::Length(N)`.
All values below map directly to `Constraint::Length` or padding counts.

| Token | Cells | Usage |
|-------|-------|-------|
| xs | 1 cell | Inline padding inside Paragraph text (leading space before content) |
| sm | 2 cells | Gap between label and input field, error message indent |
| md | 3 cells | Block height for header (progress bar row + top/bottom border) |
| lg | 3 cells | Block height for footer (key hints row + top/bottom border) |
| content | Min(10) | Content area — expands to fill remaining terminal height |
| field | 3 cells | Per-field block height (1 text row + 2 border rows) |
| error | 1 cell | Inline error line below offending field (no border, plain Paragraph) |

Exceptions: Summary page content area uses `Min(0)` to allow any number of agent rows.

## Typography

Terminal UI does not have pixel-based fonts. Typography is expressed in text attributes
applied via `Style::default().add_modifier(Modifier::*)` and color.

| Role | Modifier | Color | Usage |
|------|----------|-------|-------|
| Body | none (normal) | terminal default | Field value text, hint lines, summary agent rows |
| Label | BOLD | terminal default | Block titles (e.g. ` Project Name `, ` Tool `) |
| Heading | BOLD | Cyan | Progress indicator in header: "Step N of M — Page Title" |
| Error | none | Red | Inline error message below offending field |
| Hint | DIM | terminal default | Optional field hint line: "optional — press Enter to skip" |
| Selection | REVERSED | Cyan | Active tool selector value display: `[ claude-code ]` |

Line height: 1 terminal row per line (fixed by terminal rendering — no configuration needed).

## Color

All colors are `ratatui::style::Color` enum values. Source: established conventions in `src/commands/ui.rs`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | terminal default background | Page background — ratatui inherits terminal bg |
| Secondary (30%) | `Color::Reset` (default) | Unfocused field borders, unfocused block chrome |
| Accent (10%) | `Color::Cyan` | Active/focused field border ONLY |
| Error | `Color::Red` | Validation error border AND error message text ONLY |
| Status: ok | `Color::Green` | Agent status "idle" in summary page |
| Status: busy | `Color::Yellow` | Agent status "busy" in summary page (if shown) |
| Status: dead | `Color::Red` | Agent status "dead" in summary page (if shown) |

Accent (`Color::Cyan`) reserved for:
- Border of the currently active/focused input field
- Progress header text ("Step N of M — Page Title")

`Color::Red` reserved for:
- Border of a field that failed validation (replaces Cyan while error is active)
- Inline error message text rendered below the offending field

No other elements may use Cyan or Red.

## Copywriting Contract

### Page Titles (rendered in progress header)

| Page | Title Text |
|------|-----------|
| Page 1 | `Step 1 of N — Project Name` |
| Page 2 | `Step 2 of N — Agent Count` |
| Page 3..N+2 | `Step K of N — Agent K of M` |
| Final page | `Step N of N — Review` |

Note: `N` = total pages (2 + agent_count + 1). `K` = current step number. `M` = agent_count.

### Field Labels (Block titles)

| Field | Label |
|-------|-------|
| Project name input | ` Project Name ` |
| Agent count input | ` How Many Agents? ` |
| Role input | ` Role ` |
| Tool selector | ` Tool ` |
| Model input | ` Model ` |
| Description input | ` Description ` |

### Hint Lines (shown below optional fields, DIM style)

| Field | Hint Copy |
|-------|-----------|
| Model | `optional — press Enter to skip` |
| Description | `optional — press Enter to skip` |

### Error Messages (inline, Color::Red, shown below offending field)

| Field | Validation Failure | Error Copy |
|-------|--------------------|------------|
| Project name | Empty string | `Project name is required` |
| Agent count | Non-integer input | `Please enter a whole number (e.g. 2)` |
| Agent count | Zero or negative | `Agent count must be at least 1` |
| Role | Empty string | `Role is required` |

### Footer Key Hints (rendered in footer area, 1 row)

| Page | Footer Copy |
|------|------------|
| Page 1 (ProjectName) | `Enter: next   Ctrl+C: cancel` |
| Pages 2..N+1 | `Enter: next   Esc: back   Ctrl+C: cancel` |
| Tool selector field | `Left/Right/Tab: cycle tool   Enter: next   Esc: back   Ctrl+C: cancel` |
| Summary page | `Enter: confirm   Esc: back   Ctrl+C: cancel` |

### Summary Page

| Element | Copy |
|---------|------|
| Block title | ` Review ` |
| Project label | `Project: {project_name}` |
| Agent row format | `Agent {i}: role={role}, tool={tool}, model={model or -}, desc={desc or -}` |
| Confirm instruction | `Press Enter to confirm, Esc to go back` |

### Cancellation / Exit

| Trigger | Output | Copy |
|---------|--------|------|
| Ctrl+C (any page) | Wizard exits, returns `Ok(None)` | `Init cancelled.` (printed to stdout after terminal restore, by init.rs) |

### Empty State

Not applicable: the wizard is invoked only when no squad.yml exists. There is no list to be empty inside the wizard itself.

### Destructive Actions

None in Phase 16. The wizard collects data only — no files are written (that is Phase 17). No confirmation dialogs required.

## Interaction Contract

Source: CONTEXT.md — Implementation Decisions (locked).

### Key Bindings

| Key | Scope | Action |
|-----|-------|--------|
| `Enter` | Text input field | Submit field value; run validation; advance to next field/page on success |
| `Enter` | Tool selector | Accept current selection; advance to next field |
| `Enter` | Summary page | Confirm and return `WizardResult` to caller |
| `Esc` | Any page except Page 1 | Navigate backward one page (does NOT cancel) |
| `Esc` | Page 1 (ProjectName) | No-op (show hint in footer: "Press Ctrl+C to cancel") |
| `Ctrl+C` | Any page | Cancel wizard entirely; restore terminal; return `Ok(None)` |
| `Backspace` | Text input field | Delete last character from buffer |
| `Left` / `Right` / `Tab` | Tool selector | Cycle through tool values: claude-code → gemini-cli → antigravity → claude-code |
| Any printable char | Text input field | Append character to buffer |

### Validation Behavior

| Property | Behavior |
|----------|----------|
| When validation fires | On `Enter` press only (not on field exit, not on every keystroke) |
| On validation failure | Error message appears inline below offending field; field border turns Red; cursor stays in field |
| On validation success | Error clears; wizard advances to next field or next page |
| Tool field | No validation needed — enum selector always has a valid value selected |
| Optional fields | Empty string is valid; stored as `None` in `AgentDraft` |

### Page Transition Rules

| State | Forward (Enter) | Backward (Esc) |
|-------|-----------------|----------------|
| ProjectName | Validate non-empty → go to AgentCount | No-op |
| AgentCount | Validate positive integer → pre-allocate `agents` vec → go to AgentConfig { index: 0 } | Go to ProjectName |
| AgentConfig { index: i } where i < agent_count - 1 | Validate role non-empty → go to AgentConfig { index: i+1 } | Go to AgentConfig { index: i-1 } (or AgentCount if i==0) |
| AgentConfig { index: agent_count - 1 } | Validate role non-empty → go to Summary | Go to AgentConfig { index: agent_count - 2 } |
| Summary | Return `Ok(Some(WizardResult))` | Go to AgentConfig { index: agent_count - 1 } |

### Field Navigation Within Agent Pages

Each agent page has 4 fields rendered in order: role, tool, model, description.
All 4 are shown simultaneously on one page (no sub-stepping within the page).
`Enter` advances through fields top-to-bottom; pressing `Enter` on the last field (description) triggers page-level validation and page advance.

## Layout Contract

### Page Structure (all pages)

```
+-------------------------------------------------+
| Header — 3 rows: "Step N of M — Page Title"    |  Constraint::Length(3)
+-------------------------------------------------+
| Content — expands to fill                       |  Constraint::Min(10)
|   [fields / summary list rendered here]         |
+-------------------------------------------------+
| Footer — 2 rows: key hints                      |  Constraint::Length(2)
+-------------------------------------------------+
```

Direction: `Direction::Vertical`. Applied via `Layout::default().direction(Direction::Vertical)`.

### Field Block Dimensions

Each text input field:
- Height: `Constraint::Length(3)` (1 content row + 2 border rows)
- Width: 100% of content area width (no horizontal constraint — fills available width)
- Error line: `Constraint::Length(1)` immediately below the field block, no border

Tool selector:
- Same height as text input: `Constraint::Length(3)`
- Renders current value centered in label brackets: `[ claude-code ]`

### Summary Page Layout

```
Content area (Min(0)):
  +--------------------------+
  | Review                   |   Block with Borders::ALL, title " Review "
  |  Project: {name}         |
  |  Agent 1: role=..., ...  |
  |  Agent 2: role=..., ...  |
  |  ...                     |
  +--------------------------+

  Confirm instruction line (1 row, no border, DIM style)
```

## Component Inventory

| Component | Ratatui Widget | Notes |
|-----------|---------------|-------|
| Text input field | `Paragraph` inside `Block::default().borders(Borders::ALL)` | Buffer rendered as plain text; append `|` to buffer string for cursor indicator |
| Tool selector | `Paragraph` inside `Block::default().borders(Borders::ALL)` | Renders `[ {tool_value} ]`; not editable |
| Inline error line | `Paragraph` (no block) with `Style::default().fg(Color::Red)` | Renders `  {error_message}` with 2-space indent |
| Hint line | `Paragraph` (no block) with `Style::default().add_modifier(Modifier::DIM)` | Renders `  optional — press Enter to skip` |
| Progress header | `Paragraph` inside `Block::default().borders(Borders::ALL)` with BOLD+Cyan | Renders "Step N of M — Page Title" |
| Footer key hints | `Paragraph` (no block) | Plain text, one row |
| Summary list | `List` widget with `ListItem` rows | Same pattern as agent list in `ui.rs` |
