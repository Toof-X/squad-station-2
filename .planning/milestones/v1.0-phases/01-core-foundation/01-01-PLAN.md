**src/db/migrations/0001_initial.sql:**
```sql
CREATE TABLE IF NOT EXISTS agents (
    id          TEXT PRIMARY KEY,
    name        TEXT NOT NULL UNIQUE,
    provider    TEXT NOT NULL DEFAULT '',
    role        TEXT NOT NULL DEFAULT 'worker',
    command     TEXT NOT NULL,
    created_at  TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS messages (
    id          TEXT PRIMARY KEY,
    agent_name  TEXT NOT NULL,
    task        TEXT NOT NULL,
    status      TEXT NOT NULL DEFAULT 'pending',
    priority    TEXT NOT NULL DEFAULT 'normal',
    created_at  TEXT NOT NULL,
    updated_at  TEXT NOT NULL,
    FOREIGN KEY (agent_name) REFERENCES agents(name)
);

CREATE INDEX IF NOT EXISTS idx_messages_agent_status ON messages(agent_name, status);
CREATE INDEX IF NOT EXISTS idx_messages_priority ON messages(priority, created_at);
```

**src/tmux.rs:**
- `use std::process::Command;`
- `use anyhow::{bail, Result};`

- `pub fn send_keys_literal(target: &str, text: &str) -> Result<()>` — SAFE-02 implementation:
  - Step 1: `Command::new("tmux").args(["send-keys", "-t", target, "-l", text]).status()?` — the `-l` flag is MANDATORY, prevents special character injection
  - Check status.success(), bail with target name on failure
  - Step 2: `Command::new("tmux").args(["send-keys", "-t", target, "Enter"]).status()?` — send Enter as SEPARATE call WITHOUT `-l` so it's interpreted as a key, not literal text
  - NEVER combine text + Enter in one send-keys call

- `pub fn session_exists(session_name: &str) -> bool` — `Command::new("tmux").args(["has-session", "-t", session_name]).status().map(|s| s.success()).unwrap_or(false)` — used to check if agent/orchestrator session is alive before sending

- `pub fn launch_agent(session_name: &str, command: &str) -> Result<()>` — SAFE-03 implementation:
  - `Command::new("tmux").args(["new-session", "-d", "-s", session_name, command]).status()?`
  - Pass command directly to new-session — this runs the command as the pane's initial process, bypassing shell initialization entirely (avoids race condition from Pitfall 2)
  - Check status.success(), bail with session name on failure

- NEVER use `Command::new("sh").arg("-c")` to invoke tmux — always call tmux binary directly with explicit args to prevent shell injection

- Add `#[cfg(test)] mod tests` with unit tests that verify the Command args vectors:
  - `test_send_keys_args_have_literal_flag`: construct a mock or verify the function calls tmux with `-l` — since we can't mock Command easily, test the logic by documenting the args pattern. Alternatively, extract an `fn build_send_keys_args(target, text) -> Vec<String>` helper that returns the args and test that.

  - `test_launch_args_use_direct_command`: similar pattern for launch_agent args
