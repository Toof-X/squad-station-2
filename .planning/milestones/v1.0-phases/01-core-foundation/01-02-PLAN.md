  <name>Task 1: Implement init command — parse config, create DB, register agents, launch tmux sessions</name>
  <files>src/commands/init.rs</files>
  <action>
Replace the stub in `src/commands/init.rs` with full implementation.

**Function signature:** `pub async fn run(config_path: PathBuf, json: bool) -> anyhow::Result<()>`

**Implementation flow:**
1. Parse squad.yml: `let config = config::load_config(&config_path)?`
2. Resolve DB path: `let db_path = config::resolve_db_path(&config)?`
3. Connect to DB (creates file + runs migrations): `let pool = db::connect(&db_path).await?`
4. Register orchestrator first:
   - `db::agents::insert_agent(&pool, &config.orchestrator.name, &config.orchestrator.provider, "orchestrator", &config.orchestrator.command).await?`
   - Note: role is hardcoded to "orchestrator" regardless of config — orchestrator is structurally distinct per user decision
5. Launch orchestrator tmux session (if not already running):
   - Check `tmux::session_exists(&config.orchestrator.name)`
   - If not exists: `tmux::launch_agent(&config.orchestrator.name, &config.orchestrator.command)?`
6. Register and launch each agent — use a loop with error tracking:
   ```rust
   let mut failed: Vec<(String, String)> = vec![];
   let mut launched = 0;
   let mut skipped = 0;
   for agent in &config.agents {
       if let Err(e) = db::agents::insert_agent(&pool, &agent.name, &agent.provider, &agent.role, &agent.command).await {
           failed.push((agent.name.clone(), format!("{e:#}")));
           continue;
       }
       if tmux::session_exists(&agent.name) {
           skipped += 1;
           continue;  // Idempotent: skip already-running agents
       }
       match tmux::launch_agent(&agent.name, &agent.command) {
           Ok(()) => launched += 1,
           Err(e) => failed.push((agent.name.clone(), format!("{e:#}"))),
       }
   }
   ```
7. Output results:
   - If `json`: serialize `{ "launched": launched, "skipped": skipped, "failed": failed, "db_path": db_path }` with serde_json
   - If not json: print minimal output per user decision:
     - `"Initialized squad '{project_name}' with {launched} agent(s)"` if all succeeded
     - For each skipped: `"  - {name}: already running (skipped)"`
     - For each failed: `"  x {name}: {error}"`
   - Print DB path: `"  Database: {db_path}"`
8. Exit code: return `Err` only if ALL agents failed (including orchestrator). Partial failure prints warnings but returns Ok(()).

**Per user decisions:**
- Minimal output by default (action + result only)
- init is idempotent: re-running skips already-running agents
- Partial failure: continue launching what works, report failures

Replace the stub in `src/commands/register.rs` with full implementation.

**Function signature:** `pub async fn run(name: String, command: String, role: String, provider: String, json: bool) -> anyhow::Result<()>`

**Implementation flow:**
1. Resolve DB path — since register runs without squad.yml, use a default or require it exists:
   - Look for `squad.yml` in current directory. If exists, parse it to get DB path.
   - If no squad.yml found, check for `.squad-station-db` file or environment variable `SQUAD_STATION_DB` containing the DB path.
   - If nothing found, bail with: `"No squad.yml found in current directory. Run 'squad-station init' first, or set SQUAD_STATION_DB env var."` — this ensures register only works after init has been run at least once.
   - Alternative simpler approach: look for squad.yml, parse its project config for DB path. This is more consistent and simpler to implement.
2. Connect to DB: `let pool = db::connect(&db_path).await?`
3. Insert agent: `db::agents::insert_agent(&pool, &name, &provider, &role, &command).await?`
   - INSERT OR IGNORE means registering an existing agent name is a no-op, not an error
4. Output:
   - If `json`: `{ "registered": true, "name": name, "role": role }`
   - If not json: `"Registered agent '{name}' (role={role}, provider={provider})"`
5. Note: register does NOT launch a tmux session — it only adds to DB. User can launch the session separately. This keeps register simple and focused.
