# Requirements Archive: v1.9 Browser Visualization

**Archived:** 2026-03-22
**Status:** All requirements complete

## v1.9 Requirements

### Web Server & CLI

- [x] **SRV-01**: `squad-station browser` starts an embedded axum HTTP server that serves the React SPA from rust-embed bundled static assets
- [x] **SRV-02**: `squad-station browser` auto-opens the default system browser to the server URL after startup
- [x] **SRV-03**: Server shuts down gracefully on Ctrl+C or SIGTERM with no orphaned processes
- [x] **SRV-04**: `--port` flag allows custom port selection (default: auto-select available port)

### Real-Time Streaming

- [x] **RT-01**: axum WebSocket endpoint pushes state-change events to all connected browser clients
- [x] **RT-02**: Event-driven detection watches tmux panes and DB for state changes (agent status transitions, new/completed messages)
- [x] **RT-03**: On WebSocket connect, server sends full topology + message state snapshot as initial frame
- [x] **RT-04**: Browser auto-reconnects on WebSocket drop and re-syncs full state

### Node-Graph Visualization

- [x] **VIZ-01**: Each agent rendered as a React Flow node showing name, role, model, and live status (idle/busy/dead) with color coding
- [x] **VIZ-02**: Hierarchical auto-layout — orchestrator at top, workers below — derived from squad.yml topology
- [x] **VIZ-03**: Continuous animated arrows on edges while a message is in-flight (processing status)
- [x] **VIZ-04**: Edge labels or tooltips showing message task, priority, and timestamp

### UI Polish

- [x] **UI-01**: SPA assets bundled via rust-embed and served directly from the binary (no external files)
- [x] **UI-02**: Dark and light theme support with toggle
- [x] **UI-03**: Connection status indicator in the UI showing WebSocket state (connected/reconnecting/disconnected)
