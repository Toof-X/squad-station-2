# Milestone v1.9: Browser Visualization

**Status:** ✅ SHIPPED 2026-03-22
**Phases:** 25-28
**Total Plans:** 8

## Overview

Add `squad-station browser` command that serves a React + React Flow SPA from the binary via axum, delivering live node-graph visualization of agent topology with event-driven WebSocket streaming.

## Phases

**Depends on**: Phase 24 (v1.8 complete)
**Plans**: 2 plans

### Phase 27: Event-Driven WebSocket Streaming

**Goal**: Browser clients receive real-time state-change events pushed from the server without polling
**Depends on**: Phase 26
**Requirements**: RT-01, RT-02, RT-03, RT-04
**Plans**: 2 plans

Plans:
- [x] 27-01: Server-side WS streaming: broadcast channel, polling tasks, delta detection, snapshot builder
- [x] 27-02: Frontend WS hook, ConnectionStatus/StatusBar refactor, end-to-end verification

**Details:**
broadcast::Sender<String> with capacity 32 for multi-client fan-out. build_snapshot() for full state on connect. Delta detection (agents_changed, messages_changed) with tmux reconciliation. poll_agents() at 500ms, poll_messages() at 200ms. Subscribe before snapshot to prevent race conditions. Frontend useSquadWebSocket hook with state wipe on disconnect, auto-reconnect, presentational component refactoring.

**Depends on**: Phase 27
**Requirements**: VIZ-01, VIZ-02, VIZ-03, VIZ-04, UI-02, UI-03

Tailwind v4 dark mode via @custom-variant. React Flow colorMode sync.

## Milestone Summary

**Key Decisions:**
- axum-embed for SPA serving (handles ETag, compression, HTML5 fallback automatically)
- Feature-gated browser cargo feature (existing CLI unaffected when feature disabled)
- Event detection via polling + broadcast (not SQLite hooks — don't cross connection boundaries in WAL mode)
- Read-only SQLite pool for server (separate from single-writer pool)
- build.rs runs npm install + npm run build (single `cargo build` builds everything)
- dagre for hierarchical auto-layout (orchestrator top, workers below)
- SVG animateMotion for edge animation (GPU-accelerated, no JS animation loop)
- Tailwind v4 dark mode via @custom-variant (no config files needed)
