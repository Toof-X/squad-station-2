# Milestone v2.0: Workflow Watchdog

**Status:** ✅ SHIPPED 2026-03-24
**Phases:** 29-31
**Total Plans:** 6

## Overview

Detect stalled workflows where no agent is busy but pending/processing messages exist, and alert both the orchestrator (tmux injection) and the user (Telegram via orchestrator MCP plugin delegation).

## Phases

### Phase 29: Watchdog Core Correctness

**Goal**: Users can run `squad-station watch` and have the daemon reliably detect real stalls — deadlocks and prolonged-busy agents — without false positives, while safely coexisting with all other CLI commands
**Depends on**: Phase 28 (v1.9 shipped baseline)
**Requirements**: DETECT-01, DETECT-02, DETECT-03, DETECT-04, ALERT-01, ALERT-02, OPS-01, OPS-02, OPS-03
**Plans**: 3 plans

Plans:
- [x] 29-01: CLI flags, DB query, main.rs dispatch (foundation)
- [x] 29-02: Deadlock detection, debounce, message age, dry-run, prolonged-busy injection
- [x] 29-03: Status file writing and --status subcommand

**Details:**
Watch CLI variant with --dry-run, --status, --cooldown, --debounce, --interval, --threshold flags. DeadlockState struct with debounce counter (N consecutive cycles before trigger) and cooldown timer. NudgeState for prolonged-busy detection. Message age filtering via stall_threshold_mins. Dry-run gating at 3 injection points. WatchStatus struct with serde_json serialization for status file. --status subcommand reads status file and displays formatted health output.

### Phase 30: Telegram Integration

**Goal**: When a stall is detected, the user receives a Telegram message on their phone — delegated through the orchestrator's MCP plugin, with no HTTP client in the Rust binary
**Depends on**: Phase 29
**Requirements**: ALERT-03, ALERT-04
**Plans**: 2 plans

Plans:
- [x] 30-01: Update watchdog alert messages and orchestrator context with Telegram relay instructions
- [x] 30-02: Add channels config field, wire through launch command and YAML generation, rewrite requirements

**Details:**
Delegation architecture — watchdog injects "IMMEDIATELY USE YOUR TELEGRAM MCP PLUGIN" instruction into alert text for orchestrator to relay. Orchestrator context includes "Watchdog Alert Relay" section explaining relay protocol. channels field (Option<Vec<String>>) on AgentConfig wired through squad.yml parsing, launch command (--channels flag), and wizard YAML generation. Backward-compatible with deny_unknown_fields.

### Phase 31: End-to-End Test Coverage

**Goal**: The full tick loop behavior is verified by integration tests that run against a real SQLite DB, so correctness of deadlock detection, debounce, and deduplication is not only manually verifiable
**Depends on**: Phase 30
**Requirements**: Test coverage for all v2.0 requirements
**Plans**: 1 plan

Plans:
- [x] 31-01: CLI-level watchdog tests: --status, --dry-run, --help, flag validation, channels config

**Details:**
7 test functions in tests/test_watchdog.rs exercising the binary at CLI level. Spawn-then-kill pattern for dry-run test (daemon is infinite loop by design). Channels config test uses load_config() directly. All v2.0 requirements have at least one test covering them.

## Milestone Summary

**Key Accomplishments:**
1. Watchdog daemon with deadlock detection, prolonged-busy stall detection, configurable debounce and cooldown
2. Delegation-based Telegram alerting via orchestrator MCP plugin (zero HTTP client in Rust binary)
3. Operational tooling: --status, --dry-run, configurable flags (interval, threshold, cooldown, debounce)
4. channels config field in squad.yml wired through launch command for MCP plugin availability
5. CLI-level integration tests with full v2.0 requirement traceability

**Key Decisions:**
- Telegram delivery delegated to orchestrator MCP plugin (no reqwest/curl in binary)
- DeadlockState separate from NudgeState (merging suppresses deadlock alerts after nudge fires)
- channels as Option<Vec<String>> on AgentConfig (backward-compatible, deny_unknown_fields safe)
- Spawn-then-kill test pattern for daemon dry-run testing
- connect-per-refresh mandatory for watchdog DB access (WAL starvation prevention)
